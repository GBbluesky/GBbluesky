CSS(cascading style sheets)는 무엇인가?

css의 뜻을 직역하면 '종속형 스타일 시트'이다. css 단일로는 아무것도 할 수 없다.(내가 이해한 바로는) html문서에 결합되어 html 문서를 꾸며서 화면에 보여주는 역할을 한다. 때문에 cascading이라는 말이 붙은 것 같다.

html과 css의 작용 흐름을 보여주는 그림. 모질라의 css설명페이지에서 가져왔다.

css syntax - css는 어떻게 이루어져 있을까?

겉으로 보기에는 엄청 복잡하지만 css도 결국 하나의 언어이고 문법이 있다. css의 구문(syntax)은 다음과 같다.


css 구문를 설명한 그림. W3SCHOOLS에서 가져왔다
Selector(선택자)
선택자는 스타일을 넣고 싶은 html태그를 선택하는 역할이다. 예를 들어 h1태그로 묶은 제목의 크기를 키워주고 싶다면 h1태그를 넣는다.
Declaration(선언)
선언은 선택자에서 선택한 html태그에 어떤 속성들을 어떤 값으로 넣을지가 서술되는 자리이다. 선언은 Property(속성)과 Value(값)의 대응들로 구성된다.
Property(속성)과 Value(값)
속성은 어떤 css기능을 구현할지, 값은 속성에서 선택한 css기능의 값을 정해주는 역할이다.
